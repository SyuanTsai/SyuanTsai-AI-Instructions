# Naming Conventions

## Branches

- `main`
- `tool/<tool-name>`

## Files and directories

- Use lowercase kebab-case for Markdown docs and scripts where possible.
- Use meaningful names tied to instruction purpose.
- Keep file names in English.

## Placeholders

Use these placeholders in reusable templates:

- `{{PROJECT_NAME}}`
- `{{TECH_STACK}}`
- `{{ARCHITECTURE}}`
- `{{CODING_STYLE}}`
- `{{DOMAIN_RULES}}`
- `{{CONSTRAINTS}}`

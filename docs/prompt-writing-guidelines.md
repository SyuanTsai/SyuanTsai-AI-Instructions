# Prompt Writing Guidelines

## Principles

- Keep instructions concise and actionable.
- Prefer bullets and checklists.
- Separate mandatory rules from preferences.
- Avoid tool-specific syntax in canonical source unless needed.

## Recommended sections

1. Project context
2. Architecture and boundaries
3. Coding style
4. Testing and validation
5. Security and constraints
6. Output expectations

## Placeholder usage

Prefer placeholders to keep templates reusable:

- `{{PROJECT_NAME}}`, `{{TECH_STACK}}`, `{{ARCHITECTURE}}`
- `{{CODING_STYLE}}`, `{{DOMAIN_RULES}}`, `{{CONSTRAINTS}}`
